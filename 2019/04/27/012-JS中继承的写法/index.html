JS中继承的写法

发表于 2019-04-27 | 分类于 JS | | 阅读 次
字数统计: 571 | 阅读时长 ≈ 2

原型链继承

缺点：父类的引用属性会被子类的实例共享，子类实例不能向父类传参

构造函数继承

优点：父类引用属性不会被共享；子类构建实例可以传参
缺点：父类的方法不能复用（不能继承父类原型的方法）

组合式继承

优点：父类引用属性不会被共享；子类实例化可以传参；父类方法可以复用
缺点：两次调用父类构造函数

原型式继承

缺点：父类应用属性被共享
实际就是对传入的对象执行了一次浅拷贝

寄生式继承

缺点：不能做到函数复用

寄生组合继承

最后继承父类方法还可用以下方式实现：

  • 使用Object.create()
  • ES6 中的Object.setPrototypeOf()可以直接关联，不用手动设置constructor
